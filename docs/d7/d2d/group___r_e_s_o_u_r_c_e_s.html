Detailed Description

Loading external resources into the game.

The user is able to load external resources into the game via the LoadResources.cpp an its IonEngine::LoadResources() function. Inside the function the user can use the Asset Manager classes to load resources from the Assets folder. The Asset Managers include:

  • TextureManager
  • ModelManager
  • ShaderManager
  • ImageManager

These three can be used like so for loading assets:

LoadResources.cpp:

void IonEngine::LoadResources()
{
//---------------------------------------------------------------------------------------------------------
// Load the Models
//---------------------------------------------------------------------------------------------------------
// Model from file ( .azul format)
ModelManager::Load("modelPlane", "Models/Plane.azul");
ModelManager::Load("modelBSphere", "Models/BoundingSphere.azul");
ModelManager::Load("modelTankBody", "Models/Tank_Body.azul");
ModelManager::Load("modelTankTurret", "Models/Tank_Turret.azul");
ModelManager::Load("modelAxis", "Models/Axis.azul");
//---------------------------------------------------------------------------------------------------------
// Load the Textures
//---------------------------------------------------------------------------------------------------------
// Direct loads
TextureManager::Load("TexTankBody", "Textures/Tank_Body.tga");
TextureManager::Load("TexTankTreads", "Textures/Tank_Track.tga");
TextureManager::Load("TexTankHud", "Textures/Tank_HUD.tga");
TextureManager::Load("TexTankHudRotBODY", "Textures/Tank_HUD_Rot_BODY.tga");
TextureManager::Load("TexTankHudRotTURRET", "Textures/Tank_HUD_Rot_TURRET.tga");
TextureManager::Load("TexShellLoading", "Textures/APShell_Loading.tga");
TextureManager::Load("TexShellReady", "Textures/APShell_Ready.tga");
TextureManager::Load("TexShellWait", "Textures/Tank_HUD_Wait.tga");
TextureManager::Load("TexWinScreen", "Textures/WinScreen.tga");
TextureManager::Load("TexLoseScreen", "Textures/LoseScreen.tga");
TextureManager::Load("TexMainScreen", "Textures/MenuLogo.tga");
TextureManager::Load("TexGrass", "Textures/sandgrass.tga");
TextureManager::Load("TexFlatColor", 255, 0, 128);
//---------------------------------------------------------------------------------------------------------
// Load Images
//---------------------------------------------------------------------------------------------------------
ImageManager::Load("TankHudImage", TextureManager::Get("TexTankHud"));
ImageManager::Load("TankHudRotBODYImage", TextureManager::Get("TexTankHudRotBODY"));
ImageManager::Load("TankHudRotTURRETImage", TextureManager::Get("TexTankHudRotTURRET"));
ImageManager::Load("ShellLoadingImage", TextureManager::Get("TexShellLoading"));
ImageManager::Load("ShellReadyImage", TextureManager::Get("TexShellReady"));
ImageManager::Load("ShellWaitImage", TextureManager::Get("TexShellWait"));
ImageManager::Load("WinScreenImage", TextureManager::Get("TexWinScreen"));
ImageManager::Load("LoseScreenImage", TextureManager::Get("TexLoseScreen"));
ImageManager::Load("MainScreenImage", TextureManager::Get("TexMainScreen"));
//---------------------------------------------------------------------------------------------------------
// Load the Shaders
//---------------------------------------------------------------------------------------------------------
ShaderManager::Load("TextureFlat", "Shaders/textureFlatRender");
ShaderManager::Load("TextureLight", "Shaders/textureLightRender");
ShaderManager::Load("ColorConstant", "Shaders/colorConstantRender");
ShaderManager::Load("ColorNoTexture", "Shaders/colorNoTextureRender");
ShaderManager::Load("ColorNoTextureLight", "Shaders/colorNoTextureLightRender");
ShaderManager::Load("ColorWheel", "Shaders/colorVaryRender");
}

As you can see, the Asset Managers use their Load() functions to create and store instances of objects based off the given file path under a specified key. Later, these objects can be retrived using the Asset Managers' Get() functions with the spefified key. That would look something like this:

Frigate.cpp

Tank::Tank()
{
Vect LightColor(1.50f, 1.50f, 1.50f, 1.0f);
Vect LightPos(1.0f, 1.0f, 1.0f, 1.0f);
pGObj_Tank_Body = new GraphicsObject_TextureLight(ModelManager::Get("modelTankBody"), ShaderManager::Get("TextureLight"), TextureManager::Get("TexTankTreads"), TextureManager::Get("TexTankBody"), TextureManager::Get("TexTankBody"), LightColor, LightPos);
pGObj_Tank_Turret = new GraphicsObject_TextureLight(ModelManager::Get("modelTankTurret"), ShaderManager::Get("TextureLight"), TextureManager::Get("TexTankBody"), TextureManager::Get("TexTankBody"), TextureManager::Get("TexTankBody"), LightColor, LightPos);
}

static Texture* TextureManager::Get ( const TextureKey &  Model_Map_Key,
int  Texture_Number 
)
inlinestatic

Function returns texture pointer that is associated with a loaded Model.

This Get() function uses it's Model_Map_Key parameter and Texture_Number parameter to grab a specific texture that is associated with a model.

NOTE: In order to use this function, the texture key that is stored MUST BE THE SAME as it's file name. The Load(const TextureKey& Model_Map_Key) function will automatically do this, but if the textures are indivudually loaded with keys that are different from their file names, then this function will not work.

Parameters
Model_Map_KeyAn std::string for use as a key to access the stored model.
Texture_NumberAn integer to access the specified texture tied to the model.
Returns
pointer to texture

static void TextureManager::Load ( const TextureKey &  Model_Map_Key)
inlinestatic

Function saves all textures from a loaded model.

This Load() function creates and stores a texture pointer based off of all the textures associated with a specified model. This function accesses the model via the ModelManager and the specified Model_Map_Key string. All textures loaded via this function will use the file name of the textures as the TextureKey string for later access.

Alternatively, the user can individually call the Load(const TextureKey& Texture_Map_Key, const char* Texture_File_Path) function multiple times in order to specify the key string name of each texture.

Parameters
Model_Map_KeyAn std::string for use as a key to access the stored model.
