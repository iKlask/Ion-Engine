Detailed Description

Callbacks and set up for classes derived from Collidable.

Collidable GameObjects allow the user to register and test collision between multiple GameObjects.

Registering Collidable GameObjects happens in the Scene class. By using the SetCollisionSelf<>() and SetCollisionPair<>() functions, the user can register pairs of Collidable GameObjects that should be tested against each other for collision. More information can be found here: Scene

When two Collidable GameObjects are tested for collision and are found to be colliding, the virtual Collision() functions within the Collidable GameObjects are called. This means that the user needs to override the Collision() function in order to write the code that executes on collision.

Note
IMPORTANT: The user will need to override the Collision() function one time for every GameObject that should be tested for collision. The original Collision() function takes in a Collidable pointer as a parameter. This will also need to be overriden by the user (and in most cases, can be left empty). This original function is what will be called by default if the specific Collision() function is not specified.

Like other GameObjects, Collidables will need to be registered before collisions can be checked; they can also be deregistered to prevent collision checking while the Collidable GameObject is still alive. This is done through the CollisionRegister<>() and CollisionDeregister<>() functions.

For example, lets assume we have a bullet class. The bullet should be able to collide with a 'Frigate' GameObject and destroy itself upon collision.

Bullet.h:

class Bullet : public Updatable, public Drawable, public Alarmable, public Collidable
{
...
virtual void Collision(Collidable*);
virtual void Collision(Frigate*);
...
};

Bullet.cpp:

void Bullet::Collision(Collidable *)
{
//default collision triggered
...
}
...
void Bullet::Collision(Frigate *)
{
...
//collided with a Frigate GameObject, destroy itself
SubmitExit();
...
}
...
void Bullet::Update()
{
//calculate the new position and world matrix
...
//Update collision data with current position
this->UpdateCollisionData(world);
}
...
void Bullet::SceneEntry()
{
...
//Register for Collision
CollisionRegister<Bullet>(this);
SetColliderModel(pGObj_WireFrame_Ball->getModel());
}
...
void Bullet::SceneExit()
{
...
//Deregister for Collision
CollisionDeregister<Bullet>(this);
...
}

UserScene.cpp:

void UserScene::Initialize()
{
...
SetCollisionPair<Frigate, Bullet>();
}

First, in the header file, the bullet class derives from the Collidable class and overrides the Collision() function for both Collision(Collidable*) and Collision(Frigate*). Since Collidable virtually inherits from GameObject, bullet is also a GameObject.

Next, in the .cpp, the bullet regisisters itself for collision in the SceneEntry() with the the CollisionRegister<Bullet>(this) function. It then sets its collision model by using the function SetColliderModel(pGObj_WireFrame_Ball->getModel()). This passes a reference of the current model to the Collidable to be used for collision calculations.

In the Update() of the bullet, the function this->UpdateCollisionData(world) is called to pass the new world matrix to the Collidable to be used for collision calculations.

When the bullet actually collides with an object, the Collision() function will be called. As you can see, the defualt Collision(Collidable*) function is left empty (unless there is a desire to put something in it). In the Collision(Frigate*) function, the Collidable GameObject calss SubmitExit() in order to destroy itself upon collision with a Frigate GameObject.

In the SceneEnd() the Collidable GameObject deregisters itself with the CollisionDeregister<Bullet>(this) function. This prevents the bullet from triggering any collision while it is recycled in the BulletFactory.

Finally, within the UserScene, the SetCollisionPair<>() function is used to register 'Frigate' and 'Bullet' as a Collidable pair to be tested for collision. This means that every frame all registered Frigate GameObjects and all registered Bullet GameObjects will test if they are currently colliding.

void Collidable::Collision ( Collidable *  ptrCol)
protectedvirtual

Function that is called when collision with another Collidable GameObject is triggered.

This is a virtual function to be overriden by the user for each Collidable GameObject that is paired for Collision.

The Collision function is called every frame that the registered and paired Collidable GameObjects are colliding. To register the current Collidable GameObject for collision, see CollisionRegister() and CollisionDeregister(). To pair Collidable GameObjects for testing, see the Scene::SetCollisionPair() and Scene::SetCollisionSelf() functions.

Parameters
ptrColPointer to the Collidable GameObject that is colliding with the current Collidable GameObject.

inlineprotected

Function to deregister this Collidable GameObject from collision.

Deregistering the current Collidable GameObject removes this object from the list of Collidable's to be checked for collision each frame.

An example use of this function would go as follows:

Class::SceneExit(){
...
CollisionDeregister<Class>(*this);
}
