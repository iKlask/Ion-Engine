Detailed Description

Callbacks and set up for classes derived from Alarmable.

Alarmable GameObjects allow the user to set up alarms to execute code after a set time. This is done through the virtual alarm functions that the user can override. Each Alarmable GameObject has up to three alarms that can be overriden:

  • Alarm0(): Specified with AlarmID::ALARM_0
  • Alarm1(): Specified with AlarmID::ALARM_1
  • Alarm2(): Specified with AlarmID::ALARM_2

Since these are not pure virtual functions, an Alarmable GameObject does not have to implement all three alarms, but they cannot implemet more than three.

The AlarmableManager keeps track of all the active alarms in the scene. In order for the AlarmableManager to call the correct alarm function when its time is up, the user needs to set the alarm. This is done through the SetAlarm() function. Likewise, the user can deregister a specific alarm from the AlarmableManager with the AlarmStop() function. This ends the countdown and thus prevents the specified alarm function from being called.

For example, lets assume we have a bullet class. The bullet should only stay on the screen for a specified amount of time before it destroys/recycles itself.

Bullet.h:

class Bullet : public Updatable, public Drawable, public Alarmable, public Collidable
{
...
virtual void Alarm0() override;
...
};

Bullet.cpp:

void Bullet::Alarm0()
{
//destroy itself
SubmitExit();
}
...
void Bullet::SceneEntry()
{
...
Alarmable::SetAlarm(Bullet_Life_Time, AlarmID::ALARM_0);
...
}
...
void Bullet::SceneExit()
{
...
Alarmable::StopAlarm(AlarmID::ALARM_0);
...
}

First, in the header file, the bullet class derives from the Alarmable class and overrides the Alarm0() function. Since Alarmable virtually inherits from GameObject, bullet is also a GameObject.

Next, in the .cpp, the bullet regisisters the first alarm within the SceneEntry() function using SetAlarm(). It specifies that AlarmID::ALARM_0 should be set and it uses a predetermined 'Bullet_Life_Time' for the length of the alarm countdown.

When the alarm is activated at the Alarm0() function, it calls the SubmitExit() function.

The Alarmable GameObject then confirms that Alarm0() is deregistered from the AlarmableManager by calling StopAlarm() within the SceneExit() function (this is just incase SubmitExit() is called elsewhere while the alarm is still running).

void Alarmable::SetAlarm ( const float &  seconds,
const AlarmID &  aID 
)
protected

Function to register when a specified alarm is to be triggered.

When the SetAlarm() function is called, a float for number of seconds and an AlarmID reference is passed. The seconds parameter determines how many seconds from the current time that the alarm function be called. The aID parameter specifies which alarm function should be called. The AlarmID enum is mapped to the alarm functions like so:

  • Alarm0(): AlarmID::ALARM_0
  • Alarm1(): AlarmID::ALARM_1
  • Alarm2(): AlarmID::ALARM_2
Parameters
secondsThe number of seconds the alarm is to be set for (starting at the current time).
aIDThe ID to specify which alarm to set.

  • Alarm0(): AlarmID::ALARM_0
  • Alarm1(): AlarmID::ALARM_1
