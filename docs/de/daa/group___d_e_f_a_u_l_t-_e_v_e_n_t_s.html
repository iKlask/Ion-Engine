Detailed Description

Events that all GameObjects are registered to receive.

GameObjects will most likely be inherited through the implementation of it's child classes. More information in these modules:

  • Update events.
  • Draw events.
  • Input events.
  • Alarm events
  • Collision events

GameObjects have two pure virtual functions to be overriden by the user:

  • SceneEntry()
  • SceneExit()

These functions are to be used to register and deregister objects within the GameObject when the object is dynamically created and destroyed within the scene. This is useful when creating object factories.

GameObjects also have the SubmitEntry() and SubmitExit() functions to register commands that execute the SceneEntry() and SceneExit() functions. These submit functions can be called from other classes (like the factories).

For example, lets assume we have a bullet class. We want to be able to shoot bullets from a cannon and resuse inactive bullets. The bullet class would use the SceneEntry() and SceneExit() functions to register and deregister the bullet while the bullet factory.

Bullet.cpp:

void Bullet::SceneEntry()
{
//Register for Updatable and Drawable
Updatable::Register();
Drawable::Register();
Alarmable::SetAlarm(Bullet_Life_Time, AlarmID::ALARM_0);
//Register for Collision
CollisionRegister<Bullet>(this);
SetColliderModel(pGObj_WireFrame_Ball->getModel());
}
...
void Bullet::SceneExit()
{
Updatable::Deregister();
Drawable::Deregister();
Alarmable::StopAlarm(AlarmID::ALARM_0);
//Deregister for Collision
CollisionDeregister<Bullet>(this);
BulletFactory::RecycleBullet(this);
}
...
void Bullet::Alarm0()
{
//destroy itself
SubmitExit();
}

BulletFactory.cpp:

void BulletFactory::CreateBullet(const Matrix& rot, const Vect& pos)
{
Bullet* b;
if (recycledItems.empty())
{
b = new Bullet();
}
else
{
b = recycledItems.top();
recycledItems.pop();
}
activeItems.push_front(b);
b->Initialize(rot, pos);
b->SubmitEntry();
}
...
void BulletFactory::RecycleBullet(Bullet* bul)
{
activeItems.remove(bul);
recycledItems.push(bul);
}

As you can see, when the bullet factory puts a bullet on the screen, it will either grab a recycled bullet or create a new bullet. The bullet then has SubmitEntry() called on it so the SceneEntry() function will be called on the next update frame.

In the SceneEntry(), the bullet has an alarm set plus it is registered to be Updatable, Drawable, and Collidable.

When the alarm goes off, the SubmitExit() function is called which calls SceneExit() on the next update frame.

In SceneExit() the bullet deregisters itself from being updated, drawn, and collidable while also making sure the alarm is stopped. It then calls RecycleBullet() so the bullet instance can be reused again later.

Function Documentation

virtual void GameObject::SceneEntry ( )
protectedpure virtual

Function which is called upon a registered scene entry.

This is a pure virtual function to be overriden by the user. The SceneEntry() function will be called if the current object's SubmitEntry() function has been called elsewhere. This is used when dynamically creating objects during a scene.

This function for example might house registration function calls which need to be called on the entry to the scene.

virtual void GameObject::SceneExit ( )
protectedpure virtual

Function which is called upon a registered scene exit.

This is a pure virtual function to be overriden by the user. The SceneExit() function will be called if the current object's SubmitExit() function has been called elsewhere. This is used when dynamically destroying objects during a scene.

This function for example might house deregistration function calls which need to be called on the exit from the scene.
