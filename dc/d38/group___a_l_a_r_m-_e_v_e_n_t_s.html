Detailed Description

Callbacks and set up for classes derived from Alarmable.

Alarmable GameObjects allow the user to set up alarms to execute code after a set time. This is done through the virtual alarm functions that the user can override. Each Alarmable GameObject has up to three alarms that can be overriden:

  • Alarmable::Alarm0(): Specified with AlarmID::ALARM_0
  • Alarmable::Alarm1(): Specified with AlarmID::ALARM_1
  • Alarmable::Alarm2(): Specified with AlarmID::ALARM_2

Since these are not pure virtual functions, an Alarmable GameObject does not have to implement all three alarms, but they cannot implemet more than three.

The AlarmableManager keeps track of all the active alarms in the scene. In order for the AlarmableManager to call the correct alarm function when its time is up, the user needs to set the alarm. This is done through the Alarmable::SetAlarm() function. Likewise, the user can deregister a specific alarm from the AlarmableManager with the Alarmable::AlarmStop() function. This ends the countdown and thus prevents the specified alarm function from being called.

For example, lets assume we have a bullet class. The bullet should only stay on the screen for a specified amount of time before it destroys/recycles itself.

Bullet.h:

class Bullet : public Updatable, public Drawable, public Alarmable, public Collidable
{
...
virtual void Alarm0() override;
...
};

Bullet.cpp:

void Bullet::Alarm0()
{
//destroy itself
SubmitExit();
}
...
void Bullet::SceneEntry()
{
...
Alarmable::SetAlarm(Bullet_Life_Time, AlarmID::ALARM_0);
...
}
...
void Bullet::SceneExit()
{
...
Alarmable::StopAlarm(AlarmID::ALARM_0);
...
}

First, in the header file, the bullet class derives from the Alarmable class and overrides the Alarmable::Alarm0() function. Since Alarmable virtually inherits from GameObject, bullet is also a GameObject.

Next, in the .cpp, the bullet regisisters the first alarm within the GameObject::SceneEntry() function using Alarmable::SetAlarm(). It specifies that AlarmID::ALARM_0 should be set and it uses a predetermined 'Bullet_Life_Time' for the length of the alarm countdown.

When the alarm is activated at the Alarmable::Alarm0() function, it calls the GameObject::SubmitExit() function.

The Alarmable GameObject then confirms that Alarmable::Alarm0() is deregistered from the AlarmableManager by calling Alarmable::StopAlarm() within the GameObject::SceneExit() function (this is just incase GameObject::SubmitExit() is called elsewhere while the alarm is still running).

NOTE: The engine WILL assert if the user tries to stop an alarm that isn't set or tries to set and alarm that is already set. Make sure to avoid setting the same alarm more than once and/or stopping an alarm when it isn't set.
