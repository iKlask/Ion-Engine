Detailed Description

Callbacks and set up for classes derived from Inputable.

Inputable GameObjects allow the user to execute code when a registered key is Pressed or Released. This is done through the pure virtual functions that the user overrides: Inputable::KeyPressed() and Inputable::KeyReleased().

Everytime a registered key is pressed or released, the SingleKeyEventManager calls the Inputable::KeyPressed() or Inputable::KeyReleased() for all of the Inputables that are registered to that key. To register a key, simply use the Inputable::RegisterKey() function. Likewise, to deregister a key, and prevent the Inputable::KeyPressed() or Inputable::KeyReleased() events from triggering, use the Inputable::DeregisterKey() function.

For example, lets assume we have a frigate class. This frigate should shoot a bullet everytime the space key is pressed.

Frigate.h:

class Frigate : public Updatable, public Drawable, public Align16, public Alarmable, public Inputable, public Collidable
{
...
virtual void KeyPressed(const ION_KEY& key_in) override;
virtual void KeyReleased(const ION_KEY& key_in) override;
...
};

Frigate.cpp:

void Frigate::KeyPressed(const ION_KEY& key_in)
{
switch (key_in) {
case ION_KEY::KEY_SPACE:
BulletFactory::CreateBullet(ShipRot, ShipPos);
break;
default:
break;
}
}
void Frigate::KeyReleased(const ION_KEY& key_in)
{
//unused parameter
AZUL_UNUSED(key_in);
}
...
void Frigate::SceneEntry()
{
...
Inputable::RegisterKey(ION_KEY::KEY_SPACE, INPUT_EVENT::KEY_PRESSED);
}
...
void Frigate::SceneExit()
{
...
Inputable::DeregisterKey(ION_KEY::KEY_SPACE, INPUT_EVENT::KEY_PRESSED);
}

First, in the header file, the frigate class derives from the Inputable class and overrides the Inputable::KeyPressed() and Inputable::KeyReleased() functions. Since Inputable virtually inherits from GameObject, frigate is also a GameObject.

Next, in the .cpp, the frigate registers the key in the constructor with the Inputable::RegisterKey() function. This function takes in a key to register and which state to register it for:

  • Pressed: INPUT_EVENT::KEY_PRESSED
  • Released: INPUT_EVENT::KEY_RELEASED

Since the space key was registered for a press, the Inputable::KeyPressed() event has a switch statement and on the case that the space key is pressed, BulletFactory creates a new bullet at the current pos and current rotation.

Once the Frigate leaves the scene, the key is deregistered using the Inputable::DeregisterKey() function.

Function to deregister a specific key and it's event type (pressed or released)

This function deregisters the specified key and it's event type from being checked every frame for this Inputable GameObject. Each key can be deregistered twice from the SingleKeyEventManager; once for pressed and once for released. These can be specified by the folowing INPUT_EVENT enum types:

  • Pressed: INPUT_EVENT::KEY_PRESSED
  • Released: INPUT_EVENT::KEY_RELEASED
Parameters
key_inThe ION_KEY to be deregistered.
event_typeThe INPUT_EVENT of the key to be deregistered.
