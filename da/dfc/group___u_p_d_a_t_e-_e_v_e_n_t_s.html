Detailed Description

Callbacks and set up for classes derived from Updatable.

Updatable GameObjects allow the user to execute code once every frame. This is done through the pure virtual Updatable::Update() function that the user overrides.

The UpdatableManager keeps track of all Updatable GameObjects in the scene. In order for the UpdatableManager to call the Updatable::Update() function, the user first needs to register the current Updatable GameObject. This is done through the Updatable::Register() function. Likewise, the user can deregister the current Updatable GameObject from the UpdatableManager with the Updatable::Deregister() function. This prevents the Updatable::Update() function from being called on an Updatable GameObject that shouldn't be updated anymore.

For example, lets assume we have a bullet class. This bullet should update its position every frame after it is fired.

Bullet.h:

class Bullet : public Updatable, public Drawable, public Alarmable, public Collidable
{
...
virtual void Update() override;
...
};

Bullet.cpp:

void Bullet::Update()
{
...
//move pos forward
BulletPos += Vect(0, 0, 1) * WorldRot * BulletTransSpeed * TimeManager::GetFrameTime();
//create new world matrix based off the new position; update the model's matrix to this world matrix
Matrix world = BulletScale * WorldRot * Matrix(TRANS, BulletPos);
pGObj_WireFrame_Ball->SetWorld(world);
...
}
...
void Bullet::SceneEntry()
{
...
Updatable::Register();
...
}
...
void Bullet::SceneExit()
{
...
Updatable::Deregister();
...
}

First, in the header file, the bullet class derives from the Updatable class and overrides the Updatable::Update() function. Since Updatable virtually inherits from GameObject, bullet is also a GameObject.

Next, in the .cpp, the bullet updates its world matrix in the Updatable::Update() function every frame. It's also registered to the UpdatableManager within the GameObject::SceneEntry() function and it's deregistered from the UpdatableManager in the GameObject::SceneExit() function.
